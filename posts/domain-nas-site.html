Implementation note

从域名到 NAS：个人网站实施记录

这个网站采用公开静态主页加受保护资源入口的结构。主页部署到 Cloudflare Pages， NAS 通过 Cloudflare Tunnel 出站连接 Cloudflare，再用 Cloudflare Access 和 AList 权限做双层保护。

为什么这样设计

NAS 没有公网 IPv4 和 IPv6，不适合直接暴露服务。Tunnel 可以避免路由器端口映射； 静态主页则独立托管，即使 NAS 关机，博客和导航页也能继续访问。

域名规划

  • example.com：个人主页、博客和资源导航。
  • www.example.com：主页别名，和根域名保持一致。
  • files.example.com：Cloudflare Access 保护后的 NAS 资源入口。

上线清单

  • 把阿里云域名的 DNS nameserver 切到 Cloudflare。
  • Cloudflare Pages 连接 GitHub 仓库，部署当前静态站。
  • NAS 上用 Docker 启动 AList 和 cloudflared。
  • 在 Cloudflare Zero Trust 为 files.example.com 配置 Tunnel 和 Access 登录策略。
  • 从手机流量访问主页和资源入口，确认公网访问路径正常。

后续扩展

文章数量变多后，可以把静态 HTML 迁移到 Astro、Hugo 或 Hexo。资源入口如果需要多人使用， 再单独设计目录权限、只读账号、审计日志和备份策略。
